# M11-M15 - Early language expansion and command-system growth

## Summary

This block expanded Arqen beyond hello-world style compiler tests and started shaping reusable language commands.

## What happened

- Blend/comfort syntax experiments made the language surface less raw.
- Boolean comparison and typed symbol work expanded semantic coverage.
- Early math, command skeleton and registry ideas began to appear.
- Negative tests became more important, especially for syntax and semantic failures.

## Result

The project moved from proving that a compiler can emit something to proving that new language constructs can be added with repeatable parser/semantic/test discipline.

## Known limitation

Some exact per-milestone details from this era are intentionally summarized here rather than preserved as dozens of stale Markdown files.
