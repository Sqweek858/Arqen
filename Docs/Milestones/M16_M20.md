# M16-M20 - Pre-DX12 hardening and first DX12 bridge

## Summary

This block prepared the compiler for graphics/window/UI work and introduced the first DX12 runtime bridge.

## What happened

- Parser and command infrastructure were hardened before the graphics branch.
- Runtime, style, window and UI metadata contracts were introduced.
- M18 focused on pre-DX12 readiness, parser extraction and toolchain hardening.
- M19 introduced runtime loop/style/UI contract work.
- M20 introduced DX12 syntax/metadata, semantic checks, lowering and native clear-window smoke output.

## Result

Arqen could lower high-level metadata toward a native DX12 runtime path. This was still experimental, but it proved that the language could describe UI/window/render concepts, not only console-like output.
