# M31-M35 - UI controls and runtime state foundation

## Summary

This block closed the early UI-control bridge work and shifted the main language line toward runtime state.

## What happened

- M31 continued UI control/runtime bridge polish.
- M32/M33 focused on runtime stress and Windows x64 runtime hardening.
- M34 introduced runtime integer slots, integer operations, runtime `if`, runtime `while`, nested loops, `break` and `continue`.
- M35 introduced runtime bool and string slots, slot copy operations and runtime string printing.

## Result

Arqen moved from mostly compile-time declarations and metadata into actual runtime state and control flow.
