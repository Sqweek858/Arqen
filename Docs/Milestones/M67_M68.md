# M67-M68 - Documentation sync and root health console

M67-M68 are tooling/documentation milestones. They do not add new language syntax. They make the repository easier to verify through a central health-console entrypoint.

## M67 - Docs/tooling sync and stale path guards

- Synced the documentation with the M63-M66 tooling layout.
- Documented that `Tools/test.ps1` is the raw public test entrypoint.
- Documented that `Tools/Internal/Test/` contains internal runners.
- Added/extended stale path guards for old `Tools/Test/run_*` script references.
- Kept `Tools/Test/` empty of active public `.ps1` engines.

## M68 - Root health console

- Added root `run_me.ps1` as the central health command.
- Kept `Tools/test.ps1 -Everything -StopOnFail` as the official raw full sweep underneath.
- `run_me.ps1` adds:
  - colored console output;
  - ASCII banner;
  - preflight checks for critical tool files;
  - direct calls to clean/tool-surface/trash/repo verification;
  - official full test execution;
  - failure cause extraction from `FAIL|`, exception and log lines;
  - direct log-tail printing on failure;
  - ASCII boxed UI sections and a larger ARQEN terminal banner;
  - light startup/step transitions, disabled with `-NoAnimation`;
  - final report at `Build/Generated/run_me_report.txt`;
  - per-step logs under `Build/Logs/run_me/`.

## Normal usage

Bare `run_me.ps1` opens the interactive menu:

```powershell
.\run_me.ps1
```

For a direct standard run:

```powershell
.\run_me.ps1 -Run
```

For deeper optional surfaces:

```powershell
.\run_me.ps1 -Deep
.\run_me.ps1 -AbsolutelyEverything
.\run_me.ps1 -NoAnimation
.\run_me.ps1 -NoColor
```

`-AbsolutelyEverything` includes native/DX12 build scripts and may require external Windows build tooling. It is intentionally separate from the default path.
