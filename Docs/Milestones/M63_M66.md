# M63-M66 - Tooling cleanup, trash guards and public test entrypoint

M63-M66 are repository/tooling milestones. They do not add a major new language feature on the level of arrays, records or enums. Their purpose is to make the project safer to maintain after the M61/M62 enum-scope work.

## M63 - Tooling cleanup foundation

- Re-centered active tooling around root wrappers instead of scattered script entrypoints.
- Kept old or historical checks opt-in rather than part of the default developer path.
- Continued moving command/test infrastructure toward one predictable wrapper surface.

## M64 - Full PS1 test sweep cleanup

- Added/solidified `Tools/test.ps1 -Everything` as the combined local sweep.
- Expanded command-test results into the full report instead of hiding them behind one aggregate line.
- Kept historical validators, spec coverage validators and ExpectedIR fixtures opt-in.
- Fixed stale sample coverage so active samples compile through the current wrapper path.

## M65 - Trash and command-surface validation

- Added `Tools/Validate/Core/validate_trash.ps1` for repository garbage checks.
- Guarded against tracked generated output, patch/archive leftovers, active scripts in `Tools/Legacy/`, and active public engine scripts in `Tools/Test/`.
- Added `trash` into the normal full sweep.
- Extended tool-surface checks around root wrappers and supported switches.

## M66 - One public test entrypoint

- Kept `Tools/test.ps1` as the main public test command.
- Moved test engines to `Tools/Internal/Test/`:
  - `run_everything.ps1`
  - `run_test_slice.ps1`
- Kept `Tools/run_test_slice.ps1` as a compatibility wrapper for targeted slices.
- Required `Tools/Test/` to contain no active public `.ps1` test engines.
- Synced `verify_repo.ps1`, `validate_tool_surface.ps1`, and `validate_trash.ps1` with the internal-runner layout.

## Current testing contract

The official full local sweep is:

```powershell
.\Tools\test.ps1 -Everything -StopOnFail
```

Optional surfaces remain explicit:

```powershell
.\Tools\test.ps1 -Everything -IncludeHistoricalValidators -IncludeSpecCoverageValidators -IncludeExpectedIr -StopOnFail
.\Tools\test.ps1 -Everything -IncludeBuildScripts -StopOnFail
.\Tools\test.ps1 -Everything -IncludeScaffoldScripts -StopOnFail
```

The intended user-facing rule is simple: one complete public test command, with internal runners hidden behind wrapper scripts.
