# M69-M70 - Showcase and Visual Studio integration

M69-M70 are showcase, documentation and developer-workflow milestones. They do not introduce new public language syntax. Their purpose is to make the current Arqen surface easier to demonstrate, build and inspect from a clean workspace.

## M69 - What_I_Can_Do showcase

- Added the `What_I_Can_Do/` showcase area.
- Added `Source/`, `Build/` and `Exe/` folders for demo sources, build tooling and generated executables.
- Added console/runtime showcase samples covering current language capabilities:
  - runtime integer, boolean and string state;
  - runtime control flow;
  - functions, parameters, returns and locals;
  - arrays, records, enums and switch usage;
  - math, geometry and typed examples;
  - file I/O and command arguments.
- Added a DX12/UI showcase sample that uses the current native DX12 wrapper path.
- Added self-contained shader files under `What_I_Can_Do/Source/Shaders/` so the showcase does not depend on test-sample shader paths.
- Added `What_I_Can_Do/Build/build_all.ps1` for local showcase builds.

## M69B-M69E - Showcase build hardening

- Fixed console sample result handling so successful compiler output is not interpreted as a failed exit value.
- Switched the DX12 showcase build to the native keep-open path.
- Updated the DX12 showcase sample so it satisfies the M31A UI-controls wrapper requirements.
- Fixed DX12 shader paths by making the showcase self-contained.
- Fixed DX12 lowerer forwarding by using the internal lowering helper where required.

## M70 - Visual Studio solution shell

- Added `VisualStudio/Arqen.sln` as a contained solution for Visual Studio usage.
- Added a Makefile-style utility project, `Arqen.WhatICanDo`, that builds the `What_I_Can_Do` showcase.
- Added `VisualStudio/Trash/` as the intended location for Visual Studio build logs and local generated state.
- Added Visual Studio helper scripts under `VisualStudio/Scripts/`.
- Added a local `VisualStudio/.gitignore` to keep `.vs`, logs and local IDE artifacts out of the repository.

## M70B-M70G - Visual Studio and native-build fixes

- Corrected Visual Studio solution project-type GUIDs so solution folders open normally.
- Added Visual Studio build diagnostics and a full build log at `VisualStudio/Trash/Arqen.WhatICanDo/last_build.log`.
- Added MSVC environment bootstrap support for cases where `cl.exe` is not already available.
- Fixed the DX12 hold-time default passed by the Visual Studio wrapper.
- Fixed DX12 executable discovery/copying from `Build/EXE/` into `What_I_Can_Do/Exe/`.
- Fixed native object-output hygiene for DX12 builds by placing `.obj` and `.pdb` files under the showcase artifact directory instead of the repository root.

## M70H-M70I - README refresh

- Added `What_I_Can_Do/README.md` with explicit build and run instructions for console and DX12 demos.
- Refreshed the root `README.md` with project overview, quick start, repository layout, validation commands, Visual Studio usage and current capability summary.
- Documented that the preferred DX12 demo path is:

```text
1. Open VisualStudio\Arqen.sln.
2. Run Build > Build Solution.
3. Open What_I_Can_Do\Exe\06_dx12_ui_scene_max_metadata.exe.
```
