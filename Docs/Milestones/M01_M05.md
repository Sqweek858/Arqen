# M01-M05 - Bootstrap and first executable pipeline

## Summary

The project began as a byte-level Windows executable experiment. The early goal was not to design a full language yet, but to prove that Arqen could produce a real executable and control a minimal Windows program.

## What happened

- M1 proved manual `ExitProcess` executable construction and low-level PE layout sanity.
- M2 moved from silent exit to a visible `MessageBoxW` smoke result.
- M3/M4 explored static executable writing, template patching, strict `.arq` input reading, and early error handling.
- M5 introduced a minimal command-line compiler flow instead of only one-off emitters.

## Result

By the end of this block, Arqen had a minimal executable generation path and enough bootstrap discipline to justify moving from byte experiments toward a lexer/parser/compiler pipeline.

## Limitations kept from this era

The early artifacts were useful historically, but most are no longer active source. Current public documentation keeps the story here instead of scattering old experiment logs across the repository.
