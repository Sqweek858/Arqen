# M06-M10 - Lexer, parser, AST, codegen and M10 driver foundation

## Summary

This block turned Arqen from a byte-emitter experiment into an actual compiler/toolchain prototype.

## What happened

- M6 introduced lexer work, token output and line/column diagnostics.
- M7 introduced parser and AST output.
- M8 connected parsing to executable generation through a stricter codegen path.
- M9 expanded variables, literals, semantic checks and stricter diagnostics.
- M10 added expression handling, string concatenation and the start of the single-driver direction.

## M10 driver line

The M10/M10G line became the main compiler driver. It currently carries much more than its original scope, so it is treated as the active legacy driver until a cleaner driver v2 exists.

## Result

Arqen now had a recognizable pipeline:

```text
source .arq -> lexer -> parser -> semantic checks -> AST/IR -> backend -> executable
```
